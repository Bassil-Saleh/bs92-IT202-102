tag (all of which are<br>enclosed in a &lt;select&gt; tag to form a dropdown menu of accounts and<br>their current balance).</li></ol><div>To determine whether the user wants to perform a deposit or<br>a withdrawal, a radio-type &lt;input&gt; account type.</li><li>Retrieve the current balance of<br>the user&#39;s selected bank account from the <code>Accounts</code> table by&nbsp;using the account #<br>selected from the dropdown menu.</li><li>World account balance change = (current world account balance)<br>- (user deposit request amount)<br></li><li>User account balance change = (current user account balance)<br>+ (user deposit request amount)</li></ol><div>Expected total calculation for withdrawal:</div></div><div><ol><li>Get the ID # of<br>the world bank account from the <code>Accounts</code> table by searching for records with<br>the &quot;world&quot; account type.</li><li>Retrieve the current balance of the user&#39;s<br>selected bank account from the <code>Accounts</code> table by&nbsp;using the account # selected from<br>the dropdown menu.</li><li>If (current user account balance) &lt; (user withdraw request amount), abort<br>and send a message to the user explaining they can&#39;t perform the withdrawal<br>due to insufficient funds. Otherwise, continue to the next step.</li><li>New world account balance<br>= (current world account balance) + (user withdraw request amount)</li><li>New user account balance<br>= (current user account balance) - (user withdraw request amount)</li></ol></div><div>Two transaction records for<br>deposit:</div><div><ol><li>Calculate the expected totals for the user bank account and the world bank<br>account (see above).</li><li>Insert a record into the <code>Transactions</code> SQL table (source account =<br>user bank account, destination account = world bank account).
